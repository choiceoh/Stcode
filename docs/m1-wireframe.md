### 메시지 종류별 렌더 규칙

| codex item / event | 화면 표시 | 비고 |
|---|---|---|
| `item/agentMessage` | 🤖 + 본문 (마크다운, delta 스트리밍) | 가장 많이 보임 |
| `item/userMessage` | 🧑 + 본문 | 사용자 입력 echo |
| `item/reasoning/*Delta` | (기본 숨김, "사고 보기" 토글) | 토큰 수 많음 — 접힘 |
| `item/commandExecution` | ⚙ 박스, 명령 + 진행/출력. 완료 시 ✅, 실패 시 ❌ | stdout/stderr 폴드 가능 |
| `item/fileChange` (started→completed) | 📄 "OO 파일을 만들었어요/수정했어요" | **디프 안 보여줌 (의도)** |
| `item/mcpToolCall` | 🔌 도구 이름 + 결과 요약 | M2 |
| `item/webSearch` | 🌐 검색어 + 결과 링크 | M2 |
| `item/plan` | ☰ 체크리스트 (pending/inProgress/completed) | M2 |
| `turn/completed status=failed` | ⚠ 친화적 에러 (codexErrorInfo → 한국어) + [되돌리기] | M3 vibe layer |

## 화면 3. 승인 다이얼로그 (모달)

`item/commandExecution/requestApproval` 받으면 즉시 오버레이.

│        │  ⚙  이 명령을 실행해도 될까요?                 │          │
│        │                                                │          │
│        │  ┌──────────────────────────────────────────┐  │          │

`item/fileChange/requestApproval`은 같은 모달 패턴, 다만:
- 제목: 📄 "이 파일을 수정해도 될까요?"
- body: 파일 경로, 패치 줄 수 ("12줄 추가, 3줄 삭제") **— 디프 본문은 안 보여줌**
- 버튼 동일

## 컴포넌트 트리 (GPUI Entity)

```
AppState (Entity, 글로벌)
├── account: Option<AccountInfo>
├── current_project: Option<ProjectState>
│   ├── path: PathBuf
│   ├── thread: Option<ThreadState>
│   │   ├── id: String
│   │   ├── items: Vec<RenderedItem>
│   │   └── pending_approval: Option<ApprovalRequest>
│   └── git_repo: Option<GitState>
└── settings: Settings

MainView (Render)
├── TopBar
├── ChatPanel (uniform_list of RenderedItem)
│   ├── AgentMessageItem
│   ├── UserMessageItem
│   ├── CommandExecItem (collapsible)
│   ├── FileChangeItem (single line)
│   └── ReasoningItem (collapsed by default)
├── InputBar (multi-line text + send / stop / attach)
├── BottomBar (token usage + undo)
└── ApprovalModal (overlay, when pending_approval Some)
```

## M1 범위 (이 와이어프레임 중)

화면 1 (welcome), 화면 2 (chat — agentMessage 스트리밍 + commandExecution 박스 + fileChange 한 줄), **화면 3 (승인 다이얼로그는 M2)**, 화면 4 (인증은 일단 API 키만), 화면 5 (설정은 M3 이후).

핵심 데모: 폴더 선택 → "안녕" 보내기 → agentMessage가 한 글자씩 흘러나오면 M1 성공.

## M2~M3 범위

- M2: 승인 다이얼로그(화면 3), commandExecution 출력 스트리밍, fileChange 알림
- M3: 친화적 에러, auto git commit + 되돌리기, 설정 화면(화면 5)
