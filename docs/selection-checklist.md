# 메시지 선택 수동 체크리스트

`SelectableText`는 agent 응답과 reasoning 패널에 쓰이는 read-only 텍스트 컴포넌트다.
자동 테스트는 drag 중 선택 range/reversed 상태를 잠그고, 실제 줄바꿈 좌표와 페인트는
GPUI 렌더링 결과를 앱에서 확인한다.

## 준비

- 앱 실행: `bash scripts/build-app.sh --debug --no-codesign` 후 `dist/Stcode.app` 실행.
- 긴 답변 또는 reasoning이 있는 세션을 준비한다.
- 코드블록, 리스트, 일반 문단이 모두 있는 답변이면 좋다.

## 드래그 경계

- [ ] 한 줄 안에서 왼쪽에서 오른쪽으로 드래그한다.
  - 기대: 선택 하이라이트가 드래그한 글자 범위만 덮는다.
- [ ] 한 줄 안에서 오른쪽에서 왼쪽으로 드래그한다.
  - 기대: 선택 범위가 뒤집혀도 정확히 같은 글자 범위가 선택된다.
- [ ] 첫 줄 중간에서 시작해 다음 줄 중간까지 드래그한다.
  - 기대: 첫 줄은 시작 지점부터 줄 끝까지, 마지막 줄은 줄 시작부터 끝 지점까지 선택된다.
- [ ] 여러 줄을 가로질러 위에서 아래로 드래그한다.
  - 기대: 중간 줄은 전체 폭으로 선택되고, 앱이 멈추지 않는다.
- [ ] 여러 줄을 가로질러 아래에서 위로 드래그한다.
  - 기대: 역방향 선택에서도 동일한 범위가 선택된다.

## 영역 바깥 드래그

- [ ] 텍스트 위에서 드래그를 시작한 뒤 컴포넌트 위쪽 바깥으로 이동한다.
  - 기대: 선택 시작이 문서 처음으로 clamp된다.
- [ ] 텍스트 위에서 드래그를 시작한 뒤 컴포넌트 아래쪽 바깥으로 이동한다.
  - 기대: 선택 끝이 문서 마지막으로 clamp된다.
- [ ] 텍스트 오른쪽 여백으로 드래그한다.
  - 기대: 해당 줄 끝에 가까운 위치로 선택된다.

## 복사

- [ ] 여러 줄 선택 후 `Cmd-C`로 복사해 외부 편집기에 붙여넣는다.
  - 기대: 선택한 원문만 복사된다.
- [ ] reasoning 패널 안의 일부 텍스트를 복사한다.
  - 기대: reasoning 텍스트도 일반 메시지처럼 복사된다.
- [ ] 코드블록 일부를 선택해 복사한다.
  - 기대: 코드 문자열이 깨지지 않고 복사된다.

## 통과 기준

- drag 방향을 바꿔도 선택 범위가 튀지 않음.
- 컴포넌트 바깥으로 드래그해도 panic/crash 없음.
- multi-line selection 하이라이트가 줄 경계와 크게 어긋나지 않음.
- 복사 결과가 선택한 원문과 일치함.
